# Pruebas

## Estrategias de Pruebas

- Se han creado tests de todos los modelos para verificar que el resultado esperado en cada una de las relaciones sean correctas.

## Casos de Prueba

- Acceso a lecciones: un estudiante matriculado debe poder acceder al contenido de las lecciones, mientras que un estudiante no matriculado no debe tener acceso.

- Creación de usuario: al registrar un usuario, el sistema debe almacenar la información y permitir el inicio de sesión.

## Cobertura de Pruebas

- Las pruebas se realizan principalmente en los modelos, las vistas y la lógica. Estas pruebas se hacen en las funcionalidades más importantes

## Automatización

- Se utiliza un modelo de testing personalizado que permite comprobar errores y detectarlos de una forma sencilla.
