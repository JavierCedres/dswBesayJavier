# Mantenimiento

## Política de Mantenimiento

- Se realizarían actualizaciones entre 2 a 4 semanas

- Para el soporte se estará disponible entre semana de la mañana a la tarde

## Changelog

- Todo el registro de cambios estará en el repositorio de GitHub

## Plan de Escabilidad

- Para mejorar el servidor habría que manejar la cantidad de usuarios que puede entrar al mismo tiempo

- Dar un tiempo limitado si el usuario está ausente para poder cerrar la sesión de manera automática
